# Fitur v1.4.0 — Batch Prediction by Multiple Tickers

## Tujuan Utama
Memungkinkan sistem untuk memproses dan memprediksi banyak *ticker* saham sekaligus dalam satu kali panggilan API (*Batch*). Ini mempercepat pengembang jika ingin memuat *dashboard* atau tabel portofolio multi-saham tanpa perlu memukul (*hit*) *endpoint* API secara berulang kali.

## Mekanisme Pemrosesan yang Aman (Sequential & Fault-Tolerant)
Pada versi ini, pemrosesan batch menggunakan metode **berurutan (Sequential)** di dalam fungsi `predict_batch`.
Alurnya adalah sebagai berikut:
1. API menerima daftar kumpulan ticker.
2. Program melakukan iterasi (loop) pada setiap ticker.
3. Di dalam loop, pemrosesan dibungkus dengan metode **Try-Except**.
4. Jika ticker valid (contoh: `BBCA.JK`), model akan memprediksi rekomendasi saham dan data akan dilampirkan dengan status `success`.
5. Jika ticker invalid/error (contoh: `INVALID.TICKER` yang tidak ditemukan datanya di Yahoo Finance), program **tidak akan crash**. *Exception* akan ditangkap dan dicatat dengan status `error`. Proses akan lanjut ke saham berikutnya.

## Informasi Endpoint
- **URL**: `POST /api/v1/predict/batch`
- **Controller**: `app.api.routes.prediction.predict_stock_batch`
- **Service**: `app.services.prediction_service.predict_batch`

## Contoh Payload Request

Pengguna mengirimkan JSON berisi array `tickers` dan `period`.

```json
{
  "tickers": [
    "BBCA.JK",
    "INVALID.TICKER",
    "TLKM.JK"
  ],
  "period": "5y"
}
```

## Contoh Payload Response

## Catatan Rilis v1.4.0
- Semua interaksi masih menggunakan pemrosesan *in-memory* tanpa menyimpan file `.csv` fisik ke dalam folder `data/` secara *default*.
- Sistem sangat stabil berkat struktur `TickerResult` Pydantic yang membuat balasan JSON terstandarisasi dengan sangat rapi.
