# v1.4.1 — API Cleanup, Validation, and Consistency

## Overview

Versi **v1.4.1** adalah patch release setelah versi **v1.4.0 Batch Prediction by Multiple Tickers**.

Tujuan utama versi ini adalah memperbaiki kualitas struktur API, memperkuat validasi input, dan membuat metadata versi project lebih konsisten.

Versi ini tidak menambahkan fitur machine learning baru. Model, fitur training, file `.pkl`, dan output rekomendasi tetap mengikuti versi sebelumnya.

---

## Background

Pada versi v1.4.0, project sudah memiliki endpoint batch prediction:

```http
POST /api/v1/predict/batch
```

Namun setelah fitur batch prediction selesai, masih ada beberapa bagian kode yang perlu dirapikan, seperti:

* versi API masih hardcoded di beberapa file,
* root endpoint masih duplicate,
* validasi input ticker dan period belum terlalu ketat,
* logic normalisasi ticker belum dipusatkan dengan rapi,
* terdapat potensi bug kecil pada proses pembacaan class label model.

Karena perubahan ini bersifat cleanup dan bug fix, maka versi yang digunakan adalah **v1.4.1**.

---

## Scope of Changes

### 1. Central Configuration

Menambahkan file konfigurasi utama:

```text
app/core/config.py
```

File ini menyimpan konfigurasi umum project seperti:

```python
APP_NAME = "Stock Recommendation Prediction API"
APP_VERSION = "v1.4.1"
API_PREFIX = "/api/v1"

MODEL_VERSION_FALLBACK = "v1.0.1"

DEFAULT_PERIOD = "5y"
ALLOWED_PERIODS = {"1y", "5y", "max"}

MAX_BATCH_TICKERS = 10
```

Tujuannya adalah mengurangi hardcoded value di banyak file.

---

### 2. Main API Metadata Cleanup

File yang diperbarui:

```text
app/main.py
```

Perubahan:

* `FastAPI(title=...)` menggunakan `APP_NAME`.
* `FastAPI(version=...)` menggunakan `APP_VERSION`.
* Root endpoint `GET /` menampilkan versi terbaru.
* Endpoint `GET /health` menampilkan status API dan versi terbaru.

---

### 3. Duplicate Root Endpoint Cleanup

File yang diperbarui:

```text
app/routes/project_routes.py
```

Perubahan:

* Menghapus duplicate endpoint `GET /` dari `project_routes.py`.
* Root endpoint hanya dikelola di `app/main.py`.
* Endpoint project tetap dipertahankan, seperti:

  * `GET /project-info`
  * `GET /docs-summary`

---

### 4. Request Validation Improvement

File yang diperbarui:

```text
app/schemas/prediction_schema.py
```

Perubahan validasi:

* `period` hanya menerima nilai:

  * `1y`
  * `5y`
  * `max`

* Batch prediction dibatasi maksimal 10 ticker per request.

Contoh request valid untuk single ticker:

```json
{
  "ticker": "BBCA.JK",
  "period": "5y"
}
```

Contoh request valid untuk batch prediction:

```json
{
  "tickers": ["BBCA.JK", "TLKM.JK"],
  "period": "5y"
}
```

Contoh request tidak valid:

```json
{
  "ticker": "BBCA.JK",
  "period": "10y"
}
```

Request tersebut akan ditolak dengan status:

```http
422 Unprocessable Entity
```

---

### 5. Prediction Service Refactor

File yang diperbarui:

```text
app/services/prediction_service.py
```

Perubahan:

* Menambahkan helper function untuk normalisasi ticker.
* Single ticker otomatis dibersihkan dengan:

  * `strip()`
  * `upper()`

Contoh:

```text
" bbca.jk " → "BBCA.JK"
```

* Batch ticker akan:

  * membersihkan spasi,
  * mengubah ke uppercase,
  * menghapus duplikat,
  * mempertahankan urutan input.

Contoh input:

```json
{
  "tickers": [" bbca.jk ", "BBCA.JK", "tlkm.jk"],
  "period": "5y"
}
```

Setelah normalisasi:

```text
BBCA.JK
TLKM.JK
```

Maka `total_requested` menjadi `2`, bukan `3`.

---

### 6. Model Loader Fix

File yang diperbarui:

```text
app/services/model_loader.py
```

Perubahan:

* Memperbaiki logic iterasi pipeline step saat mengambil class label model.
* Perubahan ini mencegah potensi error ketika membaca `classes_` dari pipeline model.

## Test Cases

### 1. Single Ticker Prediction

Request:

```json
{
  "ticker": "bbca.jk",
  "period": "5y"
}
```

Expected behavior:

```text
- API berhasil melakukan prediksi
- ticker pada response menjadi BBCA.JK
- recommendation muncul
- probabilities muncul
```

---

### 2. Batch Prediction with Duplicate Ticker

Request:

```json
{
  "tickers": [" bbca.jk ", "BBCA.JK", "tlkm.jk"],
  "period": "5y"
}
```

Expected behavior:

```text
- BBCA.JK hanya diproses satu kali
- TLKM.JK ikut diproses
- total_requested bernilai 2
- response berisi total_success dan total_failed
```

---

### 3. Invalid Period

Request:

```json
{
  "ticker": "BBCA.JK",
  "period": "10y"
}
```

Expected behavior:

```text
- Request ditolak
- Status response 422 Unprocessable Entity
```

---

### 4. Batch More Than 10 Tickers

Request dengan lebih dari 10 ticker harus ditolak.

Expected behavior:

```text
- Request ditolak
- Status response 422 Unprocessable Entity
```
