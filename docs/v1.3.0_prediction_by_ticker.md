# Dokumentasi API: Prediction by Ticker (v1.3.0)

## 1. Tujuan Fitur
Fitur Prediction by Ticker (v1.3.0) bertujuan untuk menyediakan endpoint API yang memungkinkan pengguna mendapatkan rekomendasi saham (Buy/Hold/Sell) secara langsung hanya dengan menginput kode ticker saham (contoh: `BBCA.JK`). Sistem akan secara otomatis mengunduh data historis terbaru ke memori, melakukan perhitungan indikator teknikal (feature engineering), dan memasukkannya ke model machine learning untuk menghasilkan prediksi seketika (*real-time*).

## 2. Endpoint Baru
- **URL**: `POST /api/v1/predict/ticker`
- **Deskripsi**: Menghasilkan rekomendasi saham berdasarkan input ticker. Data akan diproses 100% *in-memory* tanpa menyimpan file CSV (*raw*, *clean*, maupun *features*) secara fisik, sehingga menjaga *storage* tetap bersih.

## 3. Contoh Request
```json
{
  "ticker": "BBCA.JK",
  "period": "5y"
}
```
*(Catatan: field `period` bersifat opsional, dengan nilai default `"5y"`).*

## 4. Contoh Response Sukses (200 OK)
```json
{
  "status": "success",
  "ticker": "BBCA.JK",
  "period": "5y",
  "model_version": "v1.0.1",
  "model_name": "Random Forest",
  "latest_data_date": "2026-06-04",
  "latest_close_price": 9500.0,
  "features_used": {
    "Daily_Return": 0.015,
    "MA_5": 9400.0,
    "MA_10": 9350.0,
    "RSI": 65.5,
    "Volatility": 0.012,
    "Volume_Change": 0.05
  },
  "recommendation": "Buy",
  "confidence": 0.75,
  "probabilities": {
    "Buy": 0.75,
    "Hold": 0.20,
    "Sell": 0.05
  }
}
```

## 5. Contoh Response Error (Ticker Invalid - 500 Internal Server Error)
```json
{
  "detail": {
    "status": "error",
    "message": "Prediction by ticker failed.",
    "errors": [
      {
        "field": null,
        "message": "Data untuk ticker INVALID.JK kosong.Periksa kembali kode ticker atau koneksi internet.",
        "value": null
      }
    ]
  }
}
```

## 6. Penjelasan Field Response
- `ticker`: Kode saham yang diminta oleh pengguna.
- `period`: Rentang waktu data historis yang digunakan saat *download* data dari *yfinance*.
- `model_version`: Versi dari model machine learning yang sedang digunakan (*baseline*).
- `model_name`: Algoritma model machine learning yang memproses prediksi.
- `latest_data_date`: Tanggal baris data historis paling akhir/baru yang berhasil diunduh.
- `latest_close_price`: Harga penutupan (Close) pada tanggal data terbaru.
- `features_used`: Daftar nilai metrik teknikal (*features*) pada baris data terakhir yang digunakan sebagai input ke model.
- `recommendation`: Hasil putusan/prediksi akhir (*Buy*, *Hold*, atau *Sell*).
- `confidence`: Tingkat keyakinan model terhadap prediksi tersebut (diambil dari angka `probabilities` tertinggi).
- `probabilities`: Rincian distribusi *confidence* untuk setiap label/kelas.
- `status`: Status eksekusi request API (contoh: `"success"`).

## 7. Catatan Model Baseline
Model machine learning yang dijalankan pada *backend* ini belum ditraining ulang dengan data terbaru. Model masih memakai format *baseline* rilis **v1.0.1**. Segala hasil klasifikasi sebaiknya dilihat dari kacamata eksperimen implementasi arsitektur sistem, bukan kualitas akurasi finansial nyata.

## 8. Disclaimer Investasi
Output rekomendasi `Buy`, `Hold`, atau `Sell` dari endpoint ini **bukanlah nasihat atau instruksi investasi**. Fitur ini dibuat secara murni untuk keperluan studi/pembelajaran MLOps, demonstrasi *backend API*, dan portofolio Machine Learning.

## 9. Ringkasan Hasil Testing Manual
Dari pengujian (*manual testing*) integrasi yang telah dijalankan, divalidasi bahwa:
- **`compileall` berhasil**: Semua modifikasi penambahan fitur berjalan tanpa *syntax error* Python.
- **BBCA.JK berhasil**: Sistem sukses mengunduh data, melakukan preprocessing, membuat *feature*, dan memberikan rekomendasi serta me-return respons `200 OK`.
- **Ticker valid lain berhasil**: Dicoba pada `TLKM.JK`, proses *in-memory* bekerja lancar membuktikan parameter `ticker` bersifat dinamis.
- **Ticker invalid berhasil ditangani**: Saat dicoba ticker yang salah (`INVALID.JK`), sistem tidak mengalami *crash* mendadak, melainkan memberikan response JSON terstruktur dengan HTTP Error `500` lengkap dengan penjelasan valid.
- **Endpoint lama tetap aman**: Proses modifikasi di direktori `src/` dibuat secara *backward-compatible* sehingga fungsionalitas dan *routes* lama versi 1.2.0 tidak terdampak negatif.
